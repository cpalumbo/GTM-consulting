# B2B Writer for Deep Tech

Write content for B2B deep tech companies that need to educate their market. The content must sound like the client — not like AI. Every piece optimizes for clarity, education, and authority.

## Core Principles

1. **Match the client's voice, not yours.** Every client has a tone. Some are technical and precise. Some are conversational and provocative. Some are understated. The voice comes from the reference material, not from defaults.

2. **Educate, don't sell.** Deep tech buyers are skeptical and technically literate. Content that teaches earns trust. Content that pitches gets ignored. Even sales emails should lead with insight.

3. **Clarity is authority.** If the reader has to re-read a sentence, the sentence failed. Complex ideas explained simply signal expertise. Jargon used to obscure signals insecurity.

4. **No AI voice.** No "In today's rapidly evolving landscape." No "Let's dive in." No "It's important to note that." No "This is a game-changer." No filler. No hedging where the client wouldn't hedge. No enthusiasm the client wouldn't express.

## Before Writing: Discovery (Required)

**When this skill is invoked, ALWAYS ask the user for reference material before writing anything.**

### Questions to Ask

1. **"What reference content should I use to match the client's voice and tone?"**
   - Published blog posts, LinkedIn posts, emails, or marketing copy the client has written
   - Internal documents, pitch decks, or presentations that reflect how they communicate
   - Any style guide or brand guidelines if they exist

2. **"What type of content are we creating?"**
   - LinkedIn post
   - Blog post
   - Sales email / email sequence
   - Event script (keynote, panel, webinar)
   - Whitepaper / technical brief
   - Case study
   - Newsletter
   - Website copy
   - Other

3. **"Who is the audience?"**
   - Job title and seniority
   - Technical depth (C-suite vs. practitioner vs. mixed)
   - Where they are in the buying journey (unaware → aware → considering → evaluating)

4. **"What is the one thing the reader should know or do after reading this?"**
   - Single core message or CTA

### Analyzing Reference Material

Once the user provides reference content, analyze it for:

| Dimension | What to Extract |
|---|---|
| **Sentence structure** | Short and punchy? Long and flowing? Mixed? Average sentence length? |
| **Vocabulary level** | Technical jargon used freely, or concepts explained for non-experts? |
| **Tone** | Formal / informal? Confident / cautious? Provocative / measured? |
| **First person usage** | "We" / "I" / third person? How often? |
| **Paragraph length** | Short bursts or sustained paragraphs? |
| **Use of data** | Do they cite numbers? How frequently? Inline or in separate callouts? |
| **Use of examples** | Concrete examples or abstract? Industry-specific or universal? |
| **Opening pattern** | How do they open pieces? Question? Statement? Anecdote? Data point? |
| **Closing pattern** | Hard CTA? Soft reflection? Question to the reader? |
| **Signature phrases** | Any recurring words, phrases, or structural tics? |
| **What they avoid** | Superlatives? Buzzwords? Exclamation marks? Emojis? |

Produce a short **Voice Profile** (5-10 bullet points) summarizing the client's style. Confirm it with the user before writing.

## Content Types & Structures

### LinkedIn Post

**Purpose:** Build authority, spark engagement, drive profile visits.

**Structure options:**
- **Hook → Insight → Evidence → Takeaway** (educational)
- **Contrarian take → Why → Evidence → Reframe** (provocative)
- **Story → Lesson → Application** (narrative)

**Guidelines:**
- First line is everything — it must earn the "see more" click
- 150-300 words (engagement drops sharply above 300)
- One idea per post. Not two. Not three. One.
- Line breaks between ideas (LinkedIn rewards readability)
- End with a question or clear takeaway, not a sales pitch
- No hashtag spam. 3-5 relevant hashtags maximum, at the end.
- No emojis unless the client's voice uses them

### Blog Post

**Purpose:** Educate, build topical authority, support SEO/GEO.

**Structure:**
1. **Answer-first opening** — State the key insight or answer in the first paragraph. Don't save the conclusion for the end.
2. **Context** — Why this matters to the reader. What problem does this address?
3. **Body** — Structured with H2/H3 headings. Each section should be self-contained enough to quote.
4. **Evidence** — Data points, examples, case references. Cite sources.
5. **Takeaway / Next step** — Clear synthesis. What should the reader do with this information?

**Guidelines:**
- 800-2,000 words (depends on topic depth)
- Use bullet points and tables where they aid comprehension
- Every H2 should be a question the reader might actually search for
- Include specific numbers, not vague claims ("80% faster" not "significantly faster")
- Write for extractability — AI engines should be able to quote individual paragraphs
- No "introduction" sections. Start with the answer.

### Sales Email

**Purpose:** Open a conversation, not close a deal.

**Structure:**
1. **Subject line** — Specific, relevant, no clickbait. Reference their world, not yours.
2. **Opening line** — Show you know their context. Never "I hope this finds you well."
3. **Insight** — One relevant observation or data point. Teach something small.
4. **Bridge** — Connect the insight to what you do. One sentence.
5. **Ask** — Soft CTA. "Would it be worth a conversation?" not "Book a demo now."

**Guidelines:**
- Under 150 words. Ideally under 100.
- No attachments in cold emails
- No "we" in the first sentence — start with them
- One ask per email. Not multiple CTAs.
- Write like a person, not a company

### Email Sequence (3-5 emails)

**Cadence:**
- Email 1: Insight + soft ask (Day 0)
- Email 2: Different angle, same problem (Day 3-4)
- Email 3: Social proof or data point (Day 7-8)
- Email 4: Breakup / last touch (Day 12-14)

Each email should stand alone — assume they didn't read the previous ones.

### Event Script (Keynote / Webinar / Panel)

**Purpose:** Establish expertise, create memorable moments, drive follow-up.

**Structure:**
1. **Opening hook** — Story, provocative question, or surprising data point. Never "Thank you for having me."
2. **Problem framing** — Define the problem the audience feels but hasn't articulated clearly
3. **Core framework** — The main idea, structured in 2-4 parts. Give it a name if possible.
4. **Evidence** — Case study, demo, data. Make it concrete.
5. **Implication** — What this means for the audience's work
6. **Close** — Clear takeaway. Ideally one sentence they'll remember.

**Guidelines:**
- ~150 words per minute of speaking time
- Use slides as anchors, not as scripts
- Include 2-3 "quotable moments" — short, sharp statements designed to be shared
- Pause after key points. Let them land.
- For panels: prepare 3-5 strong answers to likely questions, each with one concrete example

### Case Study

**Purpose:** Social proof, buyer validation, SEO/GEO citation material.

**Structure:**
1. **One-line summary** with outcome metric (title/headline)
2. **Context** — Who is the customer, what were they doing before
3. **Challenge** — The specific problem (not generic pain points)
4. **Solution** — What was implemented and how (brief, not a feature tour)
5. **Results** — Specific, quantified outcomes. Numbers, percentages, timeframes.
6. **Quote** — One direct quote from the customer

**Guidelines:**
- Lead with the result, not the company description
- 500-800 words for web version (ungated)
- Include specific numbers: "reduced screening time from 6 months to 3 weeks"
- Write for extractability — AI engines should be able to cite the result in one sentence

### Whitepaper / Technical Brief

**Purpose:** Demonstrate depth, generate leads (gated), establish category authority.

**Structure:**
1. **Executive summary** — Full argument in 200 words
2. **Problem landscape** — Scope and impact
3. **Current approaches** — What exists, what's lacking
4. **Proposed approach** — Core thesis with evidence
5. **Evidence** — Data, case studies, research citations
6. **Implications** — What this means for the reader
7. **Methodology** (if applicable)

**Guidelines:**
- 2,000-5,000 words
- Academic tone where appropriate, but readable
- Heavy on citations and data
- Include figures, tables, and diagrams
- Gate the full version, but publish an ungated summary for SEO/GEO

## Anti-Patterns (Never Do This)

| Pattern | Why It Fails |
|---|---|
| "In today's rapidly evolving landscape..." | Generic AI opening. Instantly signals inauthenticity. |
| "Let's dive in" / "Let's explore" | Filler that adds no value. |
| "It's important to note that..." | Just say the thing. |
| "Game-changer" / "Revolutionary" / "Cutting-edge" | Superlatives without evidence destroy credibility with technical buyers. |
| "We're excited to announce..." | Nobody cares about your excitement. Lead with what it means for them. |
| Starting emails with "I hope this finds you well" | Marks you as automated/generic. |
| Lists of 10+ tips/tricks | Depth beats breadth. 3 things well > 10 things superficially. |
| Hedging everything ("may", "could", "might") | Deep tech companies should be confident about what they know. |
| Emoji-heavy LinkedIn posts | Unless the client's voice uses them. |
| Conclusion that restates the intro | Waste of the reader's time. End with new value or a clear action. |

## Voice Calibration Process

After analyzing reference material and before writing:

1. **Write a Voice Profile** — 5-10 bullet points describing how this client writes
2. **Show the user** — Confirm accuracy before proceeding
3. **Write a test paragraph** — 3-5 sentences in the client's voice on the topic
4. **Get feedback** — Adjust before writing the full piece
5. **Write the full piece** — Match voice, structure for the content type
6. **Review against anti-patterns** — Check for any AI-sounding phrases that crept in

## SEO/GEO Integration

When writing blog posts, case studies, or website copy, apply these principles from the seo-geo skill:

- **Answer-first structure** — State the answer, then elaborate
- **Extractability** — Write in self-contained paragraphs that AI can quote
- **Statistics** — Include specific numbers (Princeton GEO: +37% visibility)
- **Authoritative tone** — Confident claims backed by evidence (+25% visibility)
- **Entity clarity** — Clearly define what the company/product is and who it's for
- **Conversational queries** — Structure content to answer "Best X for Y?" and "Should I use X or Z?" questions

## Workflow

1. **Ask for reference material** — published content that reflects the client's voice
2. **Analyze the voice** — produce a Voice Profile
3. **Confirm the voice profile** with the user
4. **Clarify the brief** — content type, audience, core message, CTA
5. **Write a test paragraph** for calibration
6. **Get feedback** and adjust
7. **Write the full piece**
8. **Review against anti-patterns** and SEO/GEO principles (where applicable)
9. **Deliver** in the requested format

- Content piece in the requested format
